写作

Hexo文章目录为blog/source/_posts，以 .md为后缀的文件，使用Markdown语法书写。个人建议可以使用webstorm中的Markdown笔记插件，Hexo默认使用hexo new [layout] <title>命令新建文章，但是我建议新手用以下方法来进行写作。

新建文章

写作完成后，在blog/source/_posts目录下新建一个以.md为后缀的文件，如test.md，然后用Notepad++打开该文件，并把之前写好的内容复制进去保存。

文章的永久链接格式title属性取自该文件名。如使用默认的永久链接格式:year/:month/:day/:title/，则生成的链接为http://yoursite.com/2015/03/21/test/。

预览

在blog目录下右键点击Git Bash Here，执行hexo s启动服务器。

在浏览器中打开http://localhost:4000，即可看到刚才写的文章。

启动服务器后，会实时监控文件，再次修改.md文件只需刷新浏览器即可看到新的效果。

生成静态文件

预览满意后，即可执行hexo g命令生成静态文件。生成的静态文件位于blog/public目录下。

为了避免遇到莫名其妙的问题，建议使用hexo clean && hexo g命令，先清除缓存文件 (db.json) 和已生成的静态文件 (public)，再生成静态文件。

添加Meta信息

创建项目

  • GitHub Github Pages有两种类型，User Pages和Project Pages，两种类型都可以。区别在于
    - **User Pages** 访问地址是二级域名`http://username.github.io`，部署的时候需要部署到`master`分支。创建项目名称为`username.github.io`，将username改为你的用户名。
- **Project Pages** 访问地址是`http://username.github.io/projectname`，需要部署到`gh-pages`分支。创建项目名称可以随意。
